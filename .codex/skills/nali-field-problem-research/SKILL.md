## When To Use
Use before adding field operations features, roadmap claims, workflow docs, or NGO/ranger/student problem statements.

## Inspect First
`docs/FIELD_PROBLEM_RESEARCH.md`, `docs/FIELD_PROBLEM_TO_NALI_FEATURE_MAP.md`, `docs/NALI_AGENTIC_GAP_ANALYSIS.md`, `src/lib/field-templates/index.ts`.

## Required Checks
Use current primary/official sources where possible: SMART, EarthRanger, ODK, KoBoToolbox, CyberTracker, GBIF, TDWG, IUCN, NASA FIRMS, GFW. Mark assumptions clearly.

## Do Not Do
Do not invent statistics or imply Indonesian institutional usage. Do not overclaim NaLI versus mature platforms.

## Verification
Each problem entry needs affected user, painful workflow, insufficiency of current tools, NaLI solution, data source, module, risk, and status.

## Expected Output
Research-backed, actionable problem-to-feature mapping.

## Token Efficiency
Search first with `rg`; browse only sources needed for changed claims.
