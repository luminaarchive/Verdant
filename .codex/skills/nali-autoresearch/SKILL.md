## When To Use
Use before source/data/dependency decisions or conservation research docs.

## Inspect First
`docs/GITHUB_RESEARCH_FOR_NALI.md`, `docs/NALI_DATA_SOURCE_MATRIX.md`, `docs/FIELD_PROBLEM_RESEARCH.md`, `docs/SEO_GOOGLE_SEARCH_READINESS.md`.

## Required Checks
Define question, source priority, search terms, evidence table, decision, fallback, and remaining uncertainty.

## Do Not Do
Do not cite low-quality sources when primary docs exist. Do not present scraped content without source URLs.

## Verification
Docs must include source URLs and classify live/scaffolded/unconfigured/unproven.

## Expected Output
Research matrix that can be audited later.

## Token Efficiency
Limit to relevant sources; store synthesis in docs instead of keeping raw pages in context.
