## When To Use
Use for large NaLI changes, phase work, or multi-surface implementation.

## Inspect First
`docs/WORKFLOW_POWER_TOOLS_RESEARCH.md`, `docs/superpowers/plans/`, `docs/CURRENT_NALI_STATE.md`.

## Required Checks
Break work into research, plan, execute, verify, document. Track live/scaffolded/unproven status.

## Do Not Do
Do not install or run external GSD automatically in this repo unless explicitly re-approved.

## Verification
Plan must name files, tests, and rollback/risk notes for DB changes.

## Expected Output
Small, reviewable phase plan and checkpoint report.

## Token Efficiency
Reuse existing docs; avoid new planning artifacts for tiny changes.
