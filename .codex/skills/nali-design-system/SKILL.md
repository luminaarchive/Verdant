## When To Use
Use before changing any UI component, route, visual layout, color, typography, or UX copy.

## Inspect First
`DESIGN.md`, affected components, `src/app/globals.css`, existing UI components.

## Required Checks
Operational density, correct icons, no nested cards, no misleading images, no fake live wording, responsive text fit.

## Do Not Do
Do not create generic AI SaaS gradients, decorative orbs, chatbot-first surfaces, or non-Indonesian animal placeholders.

## Verification
Run route-specific Playwright QA after meaningful UI changes.

## Expected Output
UI consistent with conservation-grade field intelligence.

## Token Efficiency
Read only the relevant section of DESIGN.md and affected component tree.
